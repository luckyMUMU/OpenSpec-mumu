## 目标

创建 `d:\Code\AI\OpenSpec-mumu\docs\参考\sop_GUIDE.md`，用于指导SOP文档的编写和维护，确保文档质量和一致性。

## 文档内容规划

### 1. 文档定位
- **使用对象**: 维护SOP文档的开发者/AI Agent
- **目标**: 确保SOP文档的一致性、可维护性和渐进式披露结构

### 2. 核心准则

#### 2.1 渐进式披露原则
- L1概念层：只保留概念和链接
- L2角色层：角色定义和权限
- L3流程层：工作流程和规则
- L4参考层：模板和技术细节

#### 2.2 单一来源原则
- 每个概念只在最权威的文档中详细定义
- 其他文档只保留摘要和链接
- 避免内容重复

#### 2.3 信息密度原则
- 优先使用表格、列表、代码块
- 减少描述性文字
- 提高Token效率

### 3. 文档结构规范

#### 3.1 文件组织
```
sop/
├── AGENT_SOP.md              # 入口文档
├── AGENT_SOP_COMPACT.md      # 紧凑版
├── ROLE_CHEATSHEET.md        # 角色速查
├── 01_concept_overview.md    # L1: 概念层
├── 02_role_matrix/           # L2: 角色层
├── 03_workflow/              # L3: 流程层
├── 04_reference/             # L4: 参考层
└── prompts/                  # AI提示词
```

#### 3.2 内容层级
- 父目录只保留摘要+链接
- 子目录包含详细内容
- 禁止越级包含内容

### 4. 编写规范

#### 4.1 格式规范
- 使用Markdown标准语法
- 表格优先于段落
- 代码块用于流程和示例

#### 4.2 链接规范
- 使用相对路径
- 链接文本清晰描述目标
- 保持链接有效性

#### 4.3 版本规范
- 记录版本变更
- 标注最后更新时间
- 说明主要变更内容

### 5. 维护指南

#### 5.1 更新流程
- 识别权威来源
- 修改权威文档
- 更新相关链接

#### 5.2 审核清单
- [ ] 是否符合渐进式披露
- [ ] 是否存在重复内容
- [ ] 链接是否有效
- [ ] 格式是否统一

### 6. 输出格式

创建完整的Markdown文档，包含：
- 文档目的和定位
- 核心准则详解
- 具体编写规范
- 维护指南
- 示例和模板
