# SOP Token 优化规范

## Why
SOP 文件作为 AGENT 读取对象，当前格式包含大量 ASCII 图表、装饰性元素和冗余描述，导致 token 消耗过高。需要优化为机器友好格式，提升读取效率。

## What Changes
- 移除 ASCII 图表，转换为结构化 YAML/JSON
- 简化表格格式为紧凑列表
- 移除装饰性元素（分隔线、引用块装饰）
- 移除对 AI 无用的元信息（版本历史、文档所有者、审核日期）
- 合并重复描述
- 使用 YAML 格式替代 Markdown 表格

## Impact
- Affected specs: 所有 sop/ 目录下文件
- Affected code: 无代码变更，仅文档格式优化

## ADDED Requirements

### Requirement: 机器优先格式
SOP 文件 SHALL 使用机器优先格式：
- 使用 YAML 定义结构化数据
- 使用紧凑列表替代表格
- 移除人类导向的视觉装饰

### Requirement: 信息密度最大化
SOP 文件 SHALL 最大化信息密度：
- 每行承载有效信息
- 避免重复描述
- 使用引用替代复制

### Requirement: 保留核心语义
优化后文件 SHALL 保留所有核心语义：
- 约束规则完整保留
- 流程步骤完整保留
- 契约定义完整保留

## 优化规则

### 规则 1: ASCII 图表 → YAML 结构
```yaml
# 优化前（ASCII 图表，约 200 token）
┌─────────────────────────────────────────────────────────────────────────────┐
│  阶段 1: 理解与设计                                                          │
├─────────────────────────────────────────────────────────────────────────────┤
│  步骤 1: 需求澄清                                                            │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ • 第一轮：业务背景与目标                                              │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────────────┘

# 优化后（YAML，约 30 token）
stage_1:
  name: 理解与设计
  steps:
    - id: 1
      name: 需求澄清
      rounds: [业务背景与目标, 长期目标与短期实现, 边界与约束, 风险与假设]
```

### 规则 2: 表格 → 紧凑列表
```yaml
# 优化前（表格，约 80 token）
| 检查项 | 通过条件 | 失败处理 |
|--------|----------|----------|
| 需求澄清完成 | 完成 4 轮提问，记录完整 | 返回需求澄清 |
| 长期短期区分 | 明确长期目标与短期实现 | 返回需求澄清 |

# 优化后（YAML 列表，约 30 token）
quality_gates:
  - check: 需求澄清完成
    pass: 完成 4 轮提问，记录完整
    fail: 返回需求澄清
  - check: 长期短期区分
    pass: 明确长期目标与短期实现
    fail: 返回需求澄清
```

### 规则 3: 移除装饰元素
```yaml
# 移除项：
- --- 分隔线
- > 引用块装饰
- 👉 链接装饰
- **加粗** 装饰（保留语义但简化格式）
- 版本历史表格
- 文档所有者信息
- 审核日期信息
```

### 规则 4: 合并重复描述
```yaml
# 优化前（重复描述）
### 规范先行
1. **先写规范，后写代码**：规范定义"做什么"，代码实现"怎么做"
2. **规范是唯一真理源**：代码、测试、文档必须与规范一致

### 契约协作
4. **各环节独立上下文**：不共用状态，只通过契约传递

# 优化后（紧凑格式）
constraints:
  spec_first:
    - 先写规范后写代码
    - 规范是唯一真理源
  contract:
    - 独立上下文，契约传递
```

## 文件优化优先级

| 优先级 | 文件 | 原因 |
|--------|------|------|
| P0 | AGENT_SOP.md | 入口文件，最高频读取 |
| P0 | stage-1-design.md | 包含大量 ASCII 图表 |
| P1 | p0-constraints.md | 核心约束，高频引用 |
| P1 | p1-constraints.md | 约束定义 |
| P2 | 其他 stage-*.md | 工作流文件 |
| P3 | 其他文件 | 低频读取 |
