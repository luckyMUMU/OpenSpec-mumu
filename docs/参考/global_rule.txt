1. 沟通规范
   1.1 全中文交流
   1.2 代码注释含中文说明

2. 运行环境
   2.1 操作系统：Windows
   2.2 命令行：PowerShell 兼容

3. 核心原则
   3.1 准确重于速度
   3.2 高内聚低耦合
   3.3 严禁使用强制解包（unwrap/expect）
   3.4 任务拆分到最小可执行单元

4. 编码原则
   4.1 单一职责：一个模块/类/方法只对一种变化原因负责
   4.2 开闭原则：新增能力优先通过扩展，避免修改稳定核心
   4.3 里氏替换：子类型可替换父类型且不破坏语义
   4.4 接口隔离：接口小而专，不依赖不需要的方法
   4.5 依赖倒置：业务逻辑依赖抽象，不依赖具体实现
   4.6 迪米特法则：只与直接朋友通信，减少链式访问
   4.7 逻辑层级一致：方法内步骤须处于统一抽象层级

5. 测试规范
   5.1 测试用例与代码分离
   5.2 测试设计先行

6. 安全约束
   6.1 严禁提交密钥、密码等敏感信息
