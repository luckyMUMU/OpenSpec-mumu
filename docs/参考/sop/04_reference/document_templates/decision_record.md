# 决策记录模板

**位置**: `docs/04_context_reference/decisions/YYYY-MM-DD_[topic].md`

**创建者**: 任意 Skill（通过 `RECORD_DECISION(topic, decision)` 命令）

## 使用示例

```markdown
# 数据库选型决策

## 元信息

| 字段 | 值 |
|------|------|
| 决策日期 | 2026-02-21 |
| 决策者 | sop-architecture-design |
| 触发场景 | 新项目架构设计阶段 |
| 相关文档 | docs/01_requirements/project_prd.md |

## 背景

项目需要选择主数据库，需要考虑数据量、查询模式、团队能力等因素。

## 决策选项

| 选项 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A | PostgreSQL | 成熟稳定、功能丰富 | 运维复杂度高 |
| B | MySQL | 社区活跃、文档丰富 | 高级特性较少 |
| C | MongoDB | 灵活模式、易扩展 | 事务支持有限 |

## 决策结果

**选择**: A (PostgreSQL)

**理由**: 项目需要复杂查询和事务支持，PostgreSQL 更适合。

## 影响范围

- **影响模块**: 用户模块、订单模块
- **影响文档**: architecture_design.md
- **后续行动**: 创建数据库 schema 设计

## 来源与依赖声明

| 类型 | 来源 | 说明 |
|------|------|------|
| 输入 | project_prd.md | 需求文档 |
| 依赖 | coding_principles.md | 编码原则约束 |
```

## 与其他文档的关系

| 文档类型 | 关系 | 说明 |
|----------|------|------|
| ADR | 前置 | ADR 记录架构决策背景，Decision Record 记录具体决策 |
| design.md | 引用 | design.md 可引用 Decision Record 作为设计依据 |
| source_dependency | 关联 | Decision Record 可作为来源与依赖声明的证据 |
