# PRD模板

**位置**: `docs/01_requirements/[feature]_prd.md`  
**创建者**: sop-requirement-analyst

---

## 结构

```markdown
# [功能] PRD

## 1. 概述
- 背景: [描述]
- 目标: [一句话]
- 成功指标: [指标]

## 2. 业务分析
- 价值: [描述]
- 流程: [步骤]
- 规则: [规则列表]

## 3. 用户分析
- 画像: [人群/特征/痛点]
- 场景: [触发/步骤/结果]
- 故事: [作为XX我希望XX以便XX]

## 4. 功能需求
| ID | 需求 | 优先级 | 依赖 |
|----|------|--------|------|
| FR001 | [描述] | P0 | [依赖] |

## 5. 非功能需求
- 性能: [要求]
- 安全: [要求]
- 可用性: [要求]

## 6. 验收标准
| ID | 场景 | 操作 | 预期结果 |
|----|------|------|----------|
| AC001 | [场景] | [操作] | [结果] |
```

---

## 优先级

| 级别 | 定义 |
|------|------|
| P0 | 必须有 |
| P1 | 应该有 |
| P2 | 可以有 |
| P3 | 暂不做 |

---

## 6维度分析

| 维度 | 检查项 | 结论 |
|------|--------|------|
| 业务 | 价值/流程/规则 | [结论] |
| 用户 | 画像/场景/痛点 | [结论] |
| 功能 | 范围/优先级/依赖 | [结论] |
| 技术 | 可行性/约束/集成 | [结论] |
| 风险 | 不确定性/缓解 | [结论] |
| 验收 | 标准/指标 | [结论] |
