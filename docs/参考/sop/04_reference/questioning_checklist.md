# Spec 设计阶段提问角度检查清单

## 概述

本文档提供 Spec 设计阶段的系统性提问检查清单，帮助设计者在制定技术方案时全面考虑架构一致性、ADR 合规性、性能影响、安全风险、可维护性和兼容性等关键维度。

使用方式：
1. 在 Spec 设计初期，逐项检查各维度问题
2. 对每个检查项给出明确结论（通过/待定/需处理）
3. 对"需处理"项制定具体行动计划
4. 记录检查结果作为设计文档附件

## 使用说明

### 检查时机

1. **设计初期**：在 Spec 大纲确定后，详细设计前进行初步检查
2. **设计评审前**：在提交评审前完成全面检查
3. **重大变更时**：当设计发生重大调整时重新检查

### 检查方式

1. **自查**：设计者独立完成检查清单填写
2. **交叉检查**：由其他设计者或架构师复核
3. **评审确认**：在设计评审会上确认关键检查项

### 结果处理

| 结论 | 处理方式 |
|------|----------|
| 通过 | 记录结论，继续后续流程 |
| 待定 | 标记待确认项，明确确认责任人和时间 |
| 需处理 | 制定具体处理计划，跟踪处理进度 |
