# 手动模式模板（可复制粘贴）

## 1) 强闭环模式请求模板（手动）

```markdown
ultrawork

目标：
- [一句话目标]

验收标准（必须可验证）：
- [ ] [测试/命令/截图/日志/文件路径]
- [ ] [明确的通过条件]

上下文：
- 仓库：[/absolute/or/repo-root]
- 范围：[/dir 或 file 列表]
- 约束：引用 `05_constraints/*`（如安全/验收/状态/命令）

禁止项：
- [ ] 不要引入新依赖（除非先进入 `[USER_DECISION]`）
- [ ] 不要跨 Scope 修改（如触发则进入 `[DIR_WAITING_DEP]`）

输出要求：
- 必须产出：本次 Skill 的落盘交付物（引用 `04_reference/*` 模板）
- 必须列出：未完成 TODO + 下一步建议 Skill
```

说明：
- `ultrawork` 是手动显式意图标记，用于表达“允许连续调用多个 Skill 直到形成闭环交付”。不依赖任何自动注入机制。

## 2) 快速路径请求模板（Fast Path）

```markdown
任务类型：快速路径候选

变更声明（必须同时满足）：
- [ ] 单文件
- [ ] 变更 < 30 行
- [ ] 无逻辑变更（仅格式/注释/拼写/重命名/文档）

目标文件：
- [path]

验收：
- [ ] lint/format 通过（如适用）
- [ ] 相关测试不下降（如存在）

输出：
- 仅执行 `sop-fast-path` 调用链中的下一步 Skill
```

## 3) 深度路径请求模板（Deep Path）

```markdown
任务类型：深度路径

目标：
- [一句话目标]

需求信息（若缺失则允许进入 `[WAITING_FOR_REQUIREMENTS]`）：
- 用户场景：
  - [场景1]
  - [场景2]
- 边界/非目标：
  - [非目标1]
  - [非目标2]
- 风险：
  - [风险] → [期望缓解]

验收标准：
- [ ] L1/L2/L3/L4 对应的验收门禁（引用 `05_constraints/acceptance_criteria.md`）

输出要求：
- 需求：产出 PRD/MRD/FRD（如适用）
- 设计：产出 L2（如适用）与目录级 L3 design.md
- 实现：按 Scope 改代码并验证
- 审查：产出代码审查报告
- 同步：更新索引/导航/版本信息（如适用）
```

## 4) 并行探索请求模板（先检索后实现）

```markdown
策略：先探索（只读）→ 再设计/实现

探索目标：
- [要定位的入口/模块/调用链]
- [要确认的约束/边界/失败模式]

探索输出必须包含：
- 证据：关键文件路径 + 行号范围（可点击引用）
- 结论：可执行的下一步（推荐 Skill + 输入）

实现阶段约束：
- 若探索发现信息不足/冲突 → 必须进入 `[USER_DECISION]`
```
