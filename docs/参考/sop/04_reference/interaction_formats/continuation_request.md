# 续跑与恢复请求格式

```markdown
## 续跑请求

### 当前状态
- 状态：[`[WAITING_FOR_*]` / `[DIR_WAITING_DEP]` / `[USER_DECISION]` / 其他]
- 触发原因：[一句话]

### 已完成
| 项 | 证据 | 备注 |
|---|---|---|
| [完成项] | [文件/命令输出/链接] | [备注] |

### 未完成 TODO（必须）
| TODO | 阻塞原因 | 解除条件 |
|---|---|---|
| [待办] | [原因] | [条件] |

### 需要的输入（可选）
- [缺口1]
- [缺口2]

### 建议下一步（必须）
| 推荐 Skill | 输入 | 预期输出 |
|---|---|---|
| [sop-xxx] | [关键输入] | [交付物/证据] |

### 约束（必须）
- SSOT：引用 `05_constraints/state_dictionary.md` + `05_constraints/command_dictionary.md`
- Scope：不得跨越 design.md 目录边界；跨目录依赖必须进入 `[DIR_WAITING_DEP]`
```

---

## 依赖等待续跑（DIR_WAITING_DEP）

```markdown
## 续跑请求 - 依赖等待

### 等待对象
- 依赖目录：[/dir]
- 依赖产物：[design.md / 接口契约 / 测试资产]

### 等待条件
- [ ] 依赖目录进入 `[DIR_COMPLETED]`
- [ ] 依赖目录产物可引用（路径 + 摘要）

### 唤醒后动作
- [ ] 继续当前 Scope 的实现/验证
- [ ] 触发 `sop-code-review` 复审（如涉及关键行为变更）
```

---

## 验证失败续跑（测试/门禁未通过）

```markdown
## 续跑请求 - 验证失败

### 失败证据（必须）
- 命令：[运行命令]
- 输出摘要：[关键失败点]

### 影响范围
- [影响模块/路径]

### 修复策略（最小化）
- 只修改当前 Scope
- 优先修正实现；若需调整设计/验收 → 进入 `[USER_DECISION]` 并落盘

### 回归验证清单（必须）
- [ ] 复现失败 → 通过
- [ ] 相关回归测试通过
- [ ] 关键验收标准满足（引用 `05_constraints/acceptance_criteria.md`）
```
