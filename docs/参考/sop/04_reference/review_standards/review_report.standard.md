# 审查报告质量标准

## Scope

- 适用产物：任何审查报告（设计审查/代码审查/测试审查/验收审查）
- 目的：保证审查结论可复核、可执行、可闭环

## SSOT

- 状态与命令：`05_constraints/state_dictionary.md`、`05_constraints/command_dictionary.md`
- 交互格式：`04_reference/interaction_formats/*.md`

## 严重等级与通过规则

- 🔴：缺少关键信息导致无法复核或无法执行修复，必须补齐
- 🟡：证据不足或分类不清导致返工成本高，建议补齐
- 🟢：表达优化建议，可选采纳
- 通过门槛：🔴=0 且“必检项”满足

## 必检项（Hard requirements）

- Scope 明确：被审查的对象、范围、输入版本/链接可追溯
- 问题可定位：每条问题必须包含位置（路径/章节）与描述
- 设计依据明确：每条 🔴/🟡 必须引用依据（设计/标准/红线/验收标准）
- 建议可执行：建议必须可落盘/可验证，避免抽象口号
- 结论可判定：必须明确选择“通过/需修改/僵局”，且与统计一致

## 推荐项（Soft requirements）

- 影响说明：关键问题附带影响面与风险
- 修复优先级：同级问题按优先级排序
- 复查要点：给出复查清单或重点范围

## 项目可配置项（Project knobs）

- 是否强制包含“证据链接”字段
- 是否强制包含“回滚/风险”字段（适用于高风险变更）
