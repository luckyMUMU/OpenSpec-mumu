# 阶段 3：交付与同步

> **阶段目标**: 文档同步，索引更新，完成交付

---

## 阶段概述

阶段 3 负责确保文档与代码同步，更新索引，完成最终交付。

---

## 输入契约

```yaml
# 输入：代码变更 + 审查报告
preconditions:
  required_inputs:
    - name: "code_changes"
      type: git_diff
      path: "git commit"
      validation: "代码变更必须存在且已提交"
    - name: "code_review_report"
      type: json
      path: "contracts/stage-2-code-review.json"
      validation: "审查状态必须为 passed"
    - name: "test_report"
      type: json
      path: "contracts/stage-2-test-report.json"
      validation: "测试状态必须为 passed"
  constraints:
    - "所有审查和测试必须通过"
    - "代码必须已合并到目标分支"
```

## 文档同步规范

### 必须同步的文档

| 文档 | 同步条件 | 同步内容 |
|------|----------|----------|
| README.md | 功能变更 | 功能描述、使用说明 |
| API 文档 | 接口变更 | 接口定义、参数说明 |
| CHANGELOG.md | 所有变更 | 变更记录 |
| design.md | 设计变更 | 设计更新 |

### 同步检查清单

- [ ] README.md 已更新
- [ ] API 文档已更新（如有接口变更）
- [ ] CHANGELOG.md 已更新
- [ ] 所有链接有效
- [ ] 版本号已更新

---

## 索引更新规范

### 索引文件

| 索引文件 | 描述 | 更新时机 |
|----------|------|----------|
| `sop/index.md` | SOP 入口索引 | 结构变更 |
| `docs/index.md` | 文档索引 | 文档变更 |
| `src/index.md` | 源码索引 | 模块变更 |

### 链接验证

```
链接验证规则：
- 所有内部链接必须有效
- 从入口到任意文档最短跳数 ≤ 3
- 无孤立文档（无入链的文档）
```

## 输出契约

```yaml
# 输出：完成通知 + 变更摘要
stage_id: "stage-3-deliver-sync"
version: "1.0.0"

postconditions:
  required_outputs:
    - name: "delivery_notification"
      type: json
      path: "contracts/stage-3-delivery.json"
      format: |
        {
          "delivery_status": "completed",
          "delivered_at": "时间戳",
          "summary": "交付摘要",
          "changes": ["变更清单"]
        }
    - name: "documentation_updates"
      type: files
      path: "docs/ 或相关文档路径"
      format: "更新的文档列表"

invariants:
  - "交付物必须完整（代码 + 文档 + 测试）"
  - "所有变更必须可追溯（Git Commit Hash）"
  - "文档必须与代码同步更新"
  - "用户可见变更必须有通知"
```
