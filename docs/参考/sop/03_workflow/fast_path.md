# 快速路径 (Fast Path)

**一句话定义**：适用于单文件、小范围、无逻辑变更的快速修改流程。

---

## 适用场景

### ✅ 推荐使用
- **单文件修改**：只涉及一个文件的变更
- **小范围变更**：修改行数 < 30 行
- **无逻辑变更**：不涉及业务逻辑修改
- **文档更新**：纯文档内容的增删改
- **配置调整**：配置文件的小幅调整

### ❌ 不适用场景
- 跨文件变更
- 新功能开发
- 代码重构
- API 变更
- 涉及复杂依赖关系

---

## 处理流程

```
┌─────────────────────────────────────────────────────────────┐
│                        任务开始                              │
└───────────────────────┬─────────────────────────────────────┘
                        │
                        ▼
              ┌─────────────────┐
              │  Explorer 分析  │
              │  (快速影响评估)  │
              └────────┬────────┘
                       │
                       ▼
              ┌─────────────────┐
              │   Worker 修改   │
              │  (按设计实施)    │
              └────────┬────────┘
                       │
                       ▼
              ┌─────────────────┐
              │ Librarian 更新  │
              │  (文档同步)      │
              └────────┬────────┘
                       │
                       ▼
              ┌─────────────────┐
              │   验证与完成     │
              └─────────────────┘
```

---

## 详细步骤

### 1. Explorer 快速分析

**职责**：快速评估变更影响

**工作内容**：
- 定位目标文件
- 快速扫描依赖关系
- 识别潜在风险点

**输出**：
- 影响范围简述
- 风险提示（如有）
- 修改建议

**停止点**：复述意图与影响面，待用户确认

---

### 2. Worker 执行修改

**职责**：按设计实施代码修改

**工作内容**：
- 建立代码检查点
- 执行具体修改
- 运行相关测试
- 执行质量检查

**要求**：
- 严格按需求修改
- 不引入额外变更
- 确保测试通过
- 代码符合规范

**停止点**：展示 Diff 待审

---

### 3. Librarian 更新文档

**职责**：同步更新相关文档

**工作内容**：
- 更新代码注释
- 更新相关文档
- 维护文档索引

**注意**：
- 仅更新与变更直接相关的文档
- 保持文档简洁

---

### 4. 验证与完成

**验证内容**：
- 代码功能正确
- 测试全部通过
- 质量检查通过
- 文档同步更新

**完成标准**：
- 修改符合需求
- 无引入新问题
- 文档保持一致

---

## 时间预估

| 任务类型 | 预计时间 | 主要工作 |
|----------|----------|----------|
| 修复拼写错误 | 5分钟 | Explorer 1分钟 + Worker 3分钟 + Librarian 1分钟 |
| 添加日志 | 10分钟 | Explorer 2分钟 + Worker 6分钟 + Librarian 2分钟 |
| 修改注释 | 5分钟 | Explorer 1分钟 + Worker 2分钟 + Librarian 2分钟 |
| 配置调整 | 15分钟 | Explorer 3分钟 + Worker 10分钟 + Librarian 2分钟 |
| 简单 Bug 修复 | 30分钟 | Explorer 5分钟 + Worker 20分钟 + Librarian 5分钟 |

---

## 质量标准

### 代码质量
- **准确性**：修改正确，无引入新问题
- **简洁性**：不引入不必要的变更
- **可读性**：代码清晰，易于理解

### 文档质量
- **同步性**：文档与代码保持一致
- **简洁性**：仅更新必要的内容
- **准确性**：文档描述准确

---

## 常见误区

### ❌ 避免
- 在快速路径中进行复杂重构
- 忽视测试和质量检查
- 修改不相关的代码
- 文档更新不完整

### ✅ 推荐
- 严格判断是否符合快速路径条件
- 保持修改范围最小化
- 及时运行测试
- 同步更新相关文档

---

## 快速检查清单

- [ ] 只涉及一个文件
- [ ] 修改行数 < 30 行
- [ ] 不涉及业务逻辑变更
- [ ] 无复杂依赖关系
- [ ] 可以快速测试验证

如果以上任一条件不满足，请使用 [深度路径](./deep_path.md)。
