# AI 项目通用规约 (Minimalist v5.6)

**一句话定义**：一套用于规范 AI 辅助开发的通用工作流标准。

**核心原则**：
- **准度 > 速度**：严禁跳步，重复失败立即熔断
- **文档先行**：先变更文档（标记变更），再修改代码
- **渐进披露**：按需获取信息，避免信息过载
- **少即是多**：先复用已有能力，优先改进而非新建，新建后清理过时代码

**解决的核心痛点**：
- AI 开发过程中的理解偏差和一致性问题
- 复杂任务的分拆和追踪困难
- 代码质量与文档不同步

## 核心概念

### 渐进式披露 (Progressive Disclosure)

信息按重要性分层展示，仅在需要时展示更多细节：

```
L1: 概念层 → 是什么、为什么
L2: 角色层 → 谁来做、做什么  
L3: 流程层 → 怎么做、何时做
L4: 参考层 → 具体模板
```

### 三错即停 (3-Strike Rule)

质量控制机制：
- **Strike 1**: 分析报错，执行修正
- **Strike 2**: 停止编码，审计环境，微调方案
- **Strike 3**: **熔断** - Supervisor 介入，生成报告，等待用户决策

👉 [查看三错即停详情](./03_workflow/three_strike_rule.md)

### 少即是多 (Less is More)

能力复用与优化原则：
- **先复用**：查看是否有已有的能力（Skill/Prompt/代码）
- **优先改进**：如不足够，优先改进优化已有而非新建
- **清理过时**：创造新逻辑后，验证并处理过时或相似代码

👉 [查看能力复用指南](./skills/sop-capability-reuse/SKILL.md)
