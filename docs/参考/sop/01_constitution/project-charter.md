# 项目宪章

> **规范层级**: P0 级（宪法级规范）
> **约束强度**: 不可违背，违反即熔断

---

## 愿景

构建以规范为核心的 AI 辅助开发工作流，实现：
- 规范驱动的开发流程
- 契约式协作的独立上下文
- 分层约束的质量保障
- AI 与人类的高效协同

---

## 核心价值主张

1. **规范优先**：规范是第一性产物，代码是规范的实现
2. **质量保障**：P0 级约束不可违背，确保系统安全稳定
3. **效率提升**：契约式协作减少沟通成本，提高开发效率
4. **持续演化**：轻规范到重规范的升级归档机制

---

## 技术愿景

### 核心原则

1. **Spec-first**：规范驱动 Skill，规范是唯一真理源
2. **契约隔离**：各环节独立上下文，通过契约协作
3. **分层约束**：P0 不可违背，P1-P3 渐进放宽
4. **验证独立**：验证规范是否被满足，而非验证代码

### 技术边界

- 优先使用项目已有技术栈
- 新技术引入需通过 ADR 审批
- 禁止引入功能重复的库或工具

## 规范演化规则

### 规范层级

| 层级 | 名称 | 特征 | 审批权限 |
|------|------|------|----------|
| **P0** | 工程宪章 | 不可违背，违反即熔断 | 技术委员会 |
| **P1** | 系统规范 | 跨模块约束 | 技术负责人 |
| **P2** | 模块规范 | 单模块约束 | 模块负责人 |
| **P3** | 实现规范 | 实现细节 | 自动化验证 |

### 演化流程

```
轻规范（提案）→ 确认 → 实现 → 验证 → 归档 → 升级为重规范
```
