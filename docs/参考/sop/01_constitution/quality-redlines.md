# 质量红线

> **规范层级**: P0 级（宪法级规范）
> **约束强度**: 不可违背，违反即熔断

---

## 概述

本文档定义 SOP 流程中的**质量红线**，所有约束均为 P0 级，违反将触发熔断机制。

---

## P0-SEC 安全红线

### P0-SEC-001: 禁止硬编码密钥

| 属性 | 值 |
|------|-----|
| **描述** | 禁止在代码中硬编码密钥、密码、Token 等敏感信息 |
| **验证方法** | 静态分析工具（git-secrets、truffleHog） |
| **违反处理** | 构建失败，必须立即修复 |
| **例外流程** | 无例外 |

**示例违规**：
```python
# ❌ 错误
API_KEY = "sk-xxxxxxxxxxxx"
password = "admin123"

# ✅ 正确
API_KEY = os.environ.get("API_KEY")
password = load_from_vault("password")
```

### P0-SEC-002: 禁止使用已知漏洞库

| 属性 | 值 |
|------|-----|
| **描述** | 禁止使用存在已知安全漏洞的第三方库 |
| **验证方法** | 依赖扫描工具（npm audit、Snyk） |
| **违反处理** | 构建失败，必须升级或替换 |
| **例外流程** | 无替代方案时需技术委员会审批 |

### P0-SEC-003: 禁止关闭安全校验

| 属性 | 值 |
|------|-----|
| **描述** | 禁止在生产环境关闭身份验证、授权检查、输入验证 |
| **验证方法** | 代码审查 + 安全扫描 |
| **违反处理** | 构建失败，必须修复 |
| **例外流程** | 无例外 |

## P0-QUAL 质量红线

### P0-QUAL-001: 核心模块测试覆盖率 100%

| 属性 | 值 |
|------|-----|
| **描述** | 核心业务逻辑模块单元测试覆盖率必须达到 100% |
| **验证方法** | 覆盖率工具（istanbul、coverage.py、jacoco） |
| **违反处理** | 构建失败，必须补充测试 |
| **例外流程** | 无例外 |

**核心模块定义**：
- 业务逻辑层（Service 层）
- 数据访问层（Repository 层）
- 领域模型层（Domain 层）

### P0-QUAL-002: 禁止强制解包

| 属性 | 值 |
|------|-----|
| **描述** | 禁止使用 unwrap()、expect() 等强制解包操作 |
| **验证方法** | 静态分析工具（ESLint、clippy） |
| **违反处理** | 构建失败，必须使用安全解包 |
| **例外流程** | 性能关键路径且已证明安全时可例外（需审批） |

**示例违规**：
```rust
// ❌ 错误
let value = option.unwrap();
let result = result.expect("error");

// ✅ 正确
let value = option.ok_or(Error::NotFound)?;
let result = result.map_err(|e| Error::from(e))?;
```

### P0-QUAL-003: 禁止忽略错误

| 属性 | 值 |
|------|-----|
| **描述** | 禁止忽略函数返回的错误（如 `_ = func()`） |
| **验证方法** | 静态分析工具 |
| **违反处理** | 构建失败，必须处理错误 |
| **例外流程** | 明确不需要处理时需注释说明 |

## P0-COMP 合规红线

### P0-COMP-001: 数据隐私保护

| 属性 | 值 |
|------|-----|
| **描述** | 用户数据必须符合 GDPR/个人信息保护法要求 |
| **验证方法** | 合规审计、代码审查 |
| **违反处理** | 构建失败，必须修复 |
| **例外流程** | 无例外 |

**要求**：
- 敏感数据必须加密存储
- 用户数据访问必须记录审计日志
- 数据导出必须脱敏处理

### P0-COMP-002: 许可证合规

| 属性 | 值 |
|------|-----|
| **描述** | 第三方库许可证必须符合项目合规要求 |
| **验证方法** | 许可证扫描工具 |
| **违反处理** | 构建失败，必须替换 |
| **例外流程** | 法务审批 |

**禁止许可证**：
- GPL 系列（传染性）
- AGPL（网络传染性）
- SSPL（非开源）

## P0-ARCH 架构红线

### P0-ARCH-001: 禁止循环依赖

| 属性 | 值 |
|------|-----|
| **描述** | 模块间禁止存在循环依赖 |
| **验证方法** | 依赖分析工具（dependency-cruiser、madge） |
| **违反处理** | 构建失败，必须重构 |
| **例外流程** | 无例外 |

### P0-ARCH-002: 禁止跨层调用

| 属性 | 值 |
|------|-----|
| **描述** | 禁止跳过中间层直接调用（如 Controller 直接调用 Repository） |
| **验证方法** | 架构约束工具（ArchUnit） |
| **违反处理** | 构建失败，必须修复 |
| **例外流程** | 无例外 |

**分层架构**：
```
Controller → Service → Repository
     ↓          ↓          ↓
   DTO      Domain Model  Entity
```

## 违反处理流程

```
检测到违反 P0 约束
       ↓
   构建失败
       ↓
  记录违规详情
       ↓
  通知责任人
       ↓
   修复违规
       ↓
   重新验证
       ↓
   构建通过
```
