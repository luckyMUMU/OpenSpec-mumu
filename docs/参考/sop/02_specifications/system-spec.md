## 核心业务流程

### 5 阶段工作流

```
阶段 0: 规范重量选择
    │
    ├── 评估需求复杂度
    ├── 推荐规范重量（重规范/轻规范）
    └── 输出：规范重量决策
    │
    ▼
阶段 1: 理解与设计
    │
    ├── 需求分析（BDD 场景）
    ├── 架构设计（DDD 战术）
    ├── 实现设计（design.md）
    └── 输出：设计文档 + 契约
    │
    ▼
阶段 2: 实现与验证
    │
    ├── TDD 循环执行
    ├── 约束验证
    ├── 代码审查
    └── 输出：代码变更 + 审查报告
    │
    ▼
阶段 3: 交付与同步
    │
    ├── 文档同步
    ├── 索引更新
    └── 输出：完成通知 + 变更摘要
    │
    ▼
阶段 4: 归档与演化
    │
    ├── 创建归档记录
    ├── 升级评估
    ├── 更新重规范
    └── 输出：归档记录 + CHANGELOG
```

### 规范驱动流程

```
规范（Spec）
    │
    ├── 定义"做什么"（What）
    ├── 定义约束条件（Constraints）
    └── 定义验收标准（Acceptance Criteria）
    │
    ▼
Skill（执行工具）
    │
    ├── 读取规范作为输入
    ├── 执行规范定义的任务
    ├── 生成规范要求的产物
    └── 验证规范约束是否满足
    │
    ▼
产物（Deliverables）
    │
    └── 代码、测试、文档
```

## 外部接口定义

### 输入接口

| 接口 | 来源 | 格式 | 描述 |
|------|------|------|------|
| 需求描述 | 用户 | Markdown | 功能需求描述 |
| 规范文档 | 用户 | Markdown | 规范定义文档 |
| 契约文件 | 上游环节 | YAML/JSON | 环节间契约 |

### 输出接口

| 接口 | 目标 | 格式 | 描述 |
|------|------|------|------|
| 设计文档 | 用户 | Markdown | 架构/实现设计文档 |
| 代码变更 | 代码库 | Git Diff | 代码修改记录 |
| 审查报告 | 用户 | Markdown | 代码/架构审查报告 |
| 归档记录 | 归档库 | Markdown | 归档文档 |

### 契约接口

```yaml
# 阶段契约格式
stage_id: "stage-{N}-{unique_id}"
stage_name: "阶段名称"
version: "1.0.0"

preconditions:
  required_inputs:
    - name: "input_name"
      type: "file|data"
      path: "路径"
      validation: "验证规则"

postconditions:
  required_outputs:
    - name: "output_name"
      type: "file|data"
      path: "输出路径"
      format: "格式定义"

invariants:
  - "不变式约束"
```

## 约束验证

### P0 级约束验证

| 约束 | 验证时机 | 验证方法 |
|------|----------|----------|
| 禁止硬编码密钥 | 每次提交 | 静态分析 |
| 核心模块覆盖率 100% | 每次提交 | 覆盖率工具 |
| 禁止循环依赖 | 每次提交 | 依赖分析 |

### P1-P3 级约束验证

| 约束层级 | 验证时机 | 验证方法 |
|----------|----------|----------|
| P1 级 | 每次合并 | 自动化测试 |
| P2 级 | 每次构建 | 代码审查 |
| P3 级 | 实时 | IDE 插件 |
