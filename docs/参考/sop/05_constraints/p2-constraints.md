# P2 级约束

> **约束强度**: 单模块约束，自动化验证

---

## 概述

P2 级约束定义模块级质量要求，单模块约束，变更需模块负责人审批。

---

## 代码质量约束

### P2-CODE-001: 遵循命名约定

| 属性 | 值 |
|------|-----|
| **描述** | 代码必须遵循项目命名约定 |
| **验证方法** | 代码风格检查（ESLint、Pylint） |
| **违反处理** | 警告，建议修复 |
| **例外流程** | 遗留代码可暂时例外 |

**命名约定**:
- 类名：PascalCase
- 函数名：camelCase
- 常量名：UPPER_SNAKE_CASE
- 文件名：kebab-case

### P2-CODE-002: 函数长度限制

| 属性 | 值 |
|------|-----|
| **描述** | 函数长度不应超过 50 行 |
| **验证方法** | 代码风格检查 |
| **违反处理** | 警告，建议拆分 |
| **例外流程** | 复杂算法可例外（需注释说明） |

### P2-CODE-003: 文件长度限制

| 属性 | 值 |
|------|-----|
| **描述** | 文件长度不应超过 500 行 |
| **验证方法** | 代码风格检查 |
| **违反处理** | 警告，建议拆分 |
| **例外流程** | 数据模型文件可例外 |

## 测试约束

### P2-TEST-001: 测试命名约定

| 属性 | 值 |
|------|-----|
| **描述** | 测试函数名应描述测试场景 |
| **验证方法** | 代码风格检查 |
| **违反处理** | 警告，建议重命名 |
| **例外流程** | 无例外 |

**命名格式**: `test_<功能>_<场景>_<预期结果>`

### P2-TEST-002: 测试独立性

| 属性 | 值 |
|------|-----|
| **描述** | 测试必须独立，不依赖其他测试 |
| **验证方法** | 测试运行顺序随机化 |
| **违反处理** | 警告，建议重构 |
| **例外流程** | 集成测试可例外（需标记） |
